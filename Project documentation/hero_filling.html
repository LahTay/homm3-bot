Combines all the functions inside the Finder class, so it’s not necessary to use them independently if every function needs to be used

find_attributes(h: Hero)[source]¶

Find attack, defense, knowledge and spell power of a hero. Before checking you have to enter the hero inventory.

Parameters:

h (Hero) – Hero to insert the values to

find_main_attr(h: Hero)[source]¶

Finds the name, hero_type and level of the chosen hero. You have to be inside the hero inventory.

Parameters:

h (Hero) – Hero to fill the values to

find_skills(h: Hero)[source]¶

Finds the abilities of our hero. You need to be in the hero inventory. Inserts found skills into the appropriate skill slots, if no skill is found inserts a default SecondarySkill instance. Uses difflib.get_close_matches to make sure the found skill is always correct even if tesseract finds teh text slightly wrong

Parameters:

h (Hero) – Hero to insert the values to

find_units(h: Hero)[source]¶

Combine find_units_portraits and find_units_amount and fill in the hero

Parameters:

h (Hero) – Our hero

find_units_amount()[source]¶

Finds the exact amount of certain units a hero has. It checks all the spaces from left to right, up to down and return their value. There’s no need to enter the hero inventory, as long as the units are visible on the right side of the screen.

Returns:

Number of units in hero slots. Returns 0 if no number was found

find_units_portraits()[source]¶

Finds which units our army consists of without finding their numbers. Before checking you have to enter the hero inventory

Returns:

Either returns an instance of a found unit or None if no unit was found
